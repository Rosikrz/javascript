1. kiedy powstał JavaScript .

    2. JavaScript powstał w 1995 roku.

  2. dlaczego jest to język skryptowy

    3. JavaScript jest językiem skryptowym, ponieważ jest interpretowany, a nie kompilowany. Oznacza to, że kod JavaScript jest wykonywany przez interpreter, a nie przez procesor. Interpretery są zazwyczaj wolniejsze niż kompilatory, ale mają tę zaletę, że kod JavaScript można uruchamiać na dowolnej platformie, która ma zainstalowany interpreter JavaScript.

  3. gdzie jest wykonywany JS (klient lub serwer)

    4. JavaScript jest wykonywany na kliencie, czyli w przeglądarce internetowej. Oznacza to, że kod JavaScript jest wysyłany do przeglądarki wraz ze stroną internetową i jest wykonywany przez przeglądarkę, gdy strona jest ładowana..

  4. dlaczego jest JS to bezpieczne narzędzie

    5. JavaScript jest uważany za bezpieczne narzędzie, ponieważ jest językiem interpretowanym. Oznacza to, że kod JavaScript nie może być bezpośrednio wykonywany przez procesor, ale musi być najpierw zinterpretowany przez interpreter. Interpreter sprawdza kod JavaScript pod kątem błędów i nie pozwoli na wykonanie kodu, który jest niebezpieczny.

  5. czy jest to język obiektowy , jeśli tak to dlaczego.

    6. JavaScript jest językiem obiektowym, ponieważ obsługuje obiekty i klasy. Obiekty są zmiennymi, które mogą zawierać wiele wartości, a klasy są szablonami, które definiują, jak obiekty są tworzone.

  6. wymień zalety JS:

      7. * Jest językiem skryptowym, co oznacza, że jest łatwy do nauczenia i użycia. * Jest interpretowany, co oznacza, że kod JavaScript można uruchamiać na dowolnej platformie, która ma zainstalowany interpreter JavaScript. * Jest bezpieczny, ponieważ jest językiem interpretowanym i nie może być bezpośrednio wykonywany przez procesor. * Jest obiektowy, co oznacza, że obsługuje obiekty i klasy. * Jest wszechstronny, co oznacza, że można go używać do tworzenia różnych typów aplikacji, takich jak strony internetowe, aplikacje mobilne i aplikacje desktopowe.

  7. Co to Framework::

    8. Framework to zestaw narzędzi i bibliotek, które ułatwiają tworzenie aplikacji. Frameworki zapewniają gotowe rozwiązania dla wielu wspólnych zadań, takich jak tworzenie interfejsu użytkownika, zarządzanie danymi i komunikacja z serwerem.

  8. Do aplikacji mobilnych można użyć Framework

    9. React Native * Flutter * NativeScript * Ionic * Xamarin Frameworki te ułatwiają tworzenie aplikacji mobilnych, ponieważ zapewniają gotowe rozwiązania dla wielu wspólnych zadań, takich jak tworzenie interfejsu użytkownika, zarządzanie danymi i komunikacja z serwerem.

  9. Aplikacje responsywne to

    10. Aplikacje responsywne to aplikacje, które dostosowują swój wygląd do rozmiaru ekranu urządzenia, na którym są wyświetlane. Oznacza to, że aplikacja responsywna będzie wyglądać dobrze na telefonie komórkowym, tablecie i komputerze stacjonarnym.
